# клон discord — foxy edition 🦊🐾

это современный, быстрый и функциональный клон discord, построенный на react 19, vite, express, socket.io и поддерживающий работу с базами данных postgresql/sqlite/json в реальном времени. проект включает в себя всё необходимое для полноценного общения: текстовые каналы, личные переписки, ответы на сообщения, эмодзи-реакции, голосовые и видеозвонки (с реальными webrtc потоками и интерактивными графиками), кастомизацию профилей с загрузкой аватаров/баннеров, а также гибкий ресайзинг колонок.

---

## 🛠️ стек технологий и архитектура

проект разделен на две основные части: клиентскую (фронтенд) и серверную (бэкенд).

### 🖥️ фронтенд (клиент)
* **react 19 & vite** — основа интерфейса с поддержкой современного рендеринга и мгновенной сборки.
* **react router dom** — обеспечивает роутинг и синхронизацию состояния приложения с url (например, `/channels/:serverId/:channelId`).
* **zustand** — единый менеджер состояния (стор `src/store/useStore.js`), хранящий токен, данные профиля, список друзей, серверов, историю сообщений и активный звонок.
* **react hook form + zod** — надежная валидация форм (настроек профиля) перед отправкой на бэкенд.
* **recharts** — построение графиков активности RTC-соединения (пинг) и громкости голоса в реальном времени.
* **lucide react** — современная библиотека иконок.

### ⚙️ бэкенд (сервер)
* **node.js & express** (`server.cjs`) — легкий веб-сервер для обработки rest api запросов и авторизации.
* **socket.io** — библиотека для двусторонней связи в реальном времени (синхронизация сообщений, звонков, статусов и реакций).
* **база данных (postgres / sqlite / json)** — гибкая гибридная модель бд. если задана переменная окружения postgres, используется пул подключений к pg. в качестве резервной копии используется локальный sqlite (`database.sqlite`) или json (`database.json`).

## 🌟 подробное описание функций и принципов работы

### 1. авторизация и сессии (jwt & bcrypt)
- **регистрация / вход**: при создании аккаунта (`POST /api/auth/register`) пароль хешируется с помощью библиотеки `bcryptjs` и сохраняется в бд. генерируется уникальный аватар случайного цвета и стандартный цвет акцента.
- **сессии**: при успешном входе сервер выдает json web token (jwt), который сохраняется на клиенте в `localStorage` (`discord_token`). этот токен передается в заголовках `Authorization: Bearer <token>` при каждом rest-запросе и используется для авторизации вебсокет-соединений.

### 2. авто-миграции и гибридная база данных
- при старте `server.cjs` автоматически проверяет наличие колонок в таблице `users` (например, `display_name`, `banner_url`, `avatar_url`) и выполняет альтер-запросы `ALTER TABLE users ADD COLUMN IF NOT EXISTS ...` для динамического обновления схемы бд без потери данных.
- бэкенд умеет прозрачно переключаться между postgresql (если в системе есть переменная `DATABASE_URL`) и локальным sqlite/json. интерфейс методов `db.*` полностью одинаков для всех баз данных.

### 3. текстовый чат, ответы и эмодзи-реакции
- **история сообщений**: сообщения подгружаются при клике на текстовый канал или личную переписку через `GET /api/messages/:channelId`.
- **ответы на сообщения (Replies)**: при выборе действия «ответить» в тулбаре сообщения, айди исходного сообщения передается в поле `replyToId`. в бд оно связывается с колонкой `reply_to_id`. в интерфейсе рендерится превью цитируемого текста с красивой изогнутой линией.
- **эмодзи-реакции (Reactions)**: пользователи могут ставить реакции на сообщения. данные записываются в таблицу `message_reactions` (связка `message_id`, `username`, `emoji`). через сокеты событие `reaction_update` рассылается всем подключенным пользователям, обновляя счетчики и списки отреагировавших в реальном времени.

### 4. голосовые и видеозвонки (webrtc & websocket)
- **подключение**: при входе в голосовой канал сервер создает объект активного звонка. вебсокеты транслируют список участников звонка, состояние их микрофонов, звука, камеры и стрима.
- **фоновая работа звонка**: звонок больше **не сбрасывается** при переходе по каналам или серверам. это достигается за счет рендеринга невидимого компонента `VoiceCallManager.jsx` на уровне корня приложения (`App.jsx`). панель управления звонком отображается внизу левого сайдбара, позволяя мьютить звук и отключаться из любого места в приложении.
- **webrtc трансляция**: поддерживается захват реальной камеры (`navigator.mediaDevices.getUserMedia`) и экрана компьютера (`navigator.mediaDevices.getDisplayMedia`). локальный поток рендерится в теге `<video>` и передается другим участникам через состояние `localStream`.
- **графики активности (recharts)**:
  - *ping*: динамический график, отображающий текущую задержку. при включении стрима экрана задержка реалистично возрастает, симулируя нагрузку.
  - *voice activity*: график уровня громкости участников. при разговоре вокруг аватарки пользователя в гриде звонка пульсирует зеленый ободок.

### 5. кастомизация профиля, аватарки и баннеры
- **загрузка файлов**: в настройках профиля (`UserSettings.jsx`) пользователь может загрузить аватарку или фоновый баннер.
- **кодирование в base64**: файлы читаются на клиенте через `FileReader` как data url (base64-строка) и отправляются в базу данных. это позволяет хранить изображения прямо в бд (тип `TEXT`) без необходимости подключать внешние s3-хранилища.
- **отображение баннера**: в `UserProfileModal.jsx` баннер рендерится как фоновое изображение с помощью свойства `backgroundSize: 'cover'`. если баннер не загружен, используется выбранный пользователем цвет акцента.

### 6. интерактивный ресайзинг (resizable layout)
- разделители (`layout-resizer`) между левым сайдбаром, чатом и правым списком участников поддерживают перетаскивание мышью.
- состояние ширины колонок (`sidebarWidth` и `membersWidth`) регулируется через события `onMouseDown`, `mousemove` и `mouseup` с установкой жестких лимитов (`200px - 400px`), что предотвращает поломку верстки на любых экранах.

## 🚀 инструкции по развертыванию и запуску

### 1. установка зависимостей
установите библиотеки с флагом peer-deps, так как используются новейшие пакеты react:
```bash
npm install --legacy-peer-deps
```

### 2. настройка базы данных (опционально)
по умолчанию проект создаст локальную базу данных `database.sqlite` в корневой директории.
если вы хотите использовать postgresql, задайте переменную окружения `DATABASE_URL` (например, в файле `.env` или через панель хостинга railway/heroku):
```
DATABASE_URL=postgres://user:password@host:port/database
```

### 3. запуск в режиме разработки
запустите одновременно сервер бэкенда и фронтенд:
```bash
npm run dev
```

### 4. компиляция для продакшена
соберите оптимизированный клиентский бандл:
```bash
npm run build
```
после сборки файлы попадут в папку `dist/`, откуда экспресс-сервер будет отдавать их как статику.

### 5. запуск в качестве десктопного приложения (electron)
проект можно запустить как настоящее настольное приложение (подобно официальному клиенту discord):
```bash
npm run electron:start
```
- **как это работает**: electron создает отдельное бесшовное окно с интеграцией node.js. при запуске он пытается проверить, запущен ли локальный сервер на порту `3000`. если он запущен, приложение откроет локальную версию для разработки. если локальный сервер не запущен, electron автоматически подключится к удаленному продакшен-серверу в сети (`https://furrdis.up.railway.app/`), работая как полноценный тонкий клиент.
